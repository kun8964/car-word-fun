## 1. Product Positioning

把“车车大联盟”从车型图鉴升级成一个幼儿学习娱乐网站。核心不是让孩子背车型，而是通过可爱的磨砂树脂车车、闯关、收集、声音和视差场景，让孩子自然学习:

- 车辆认知: 车、巴士、卡车、工程车、赛车、摩托、飞机等。
- 语言启蒙: 中文名称、英文单词、首字母、简单拼写。
- 观察能力: 颜色、形状、大小、轮子数量、用途分类。
- 逻辑能力: 配对、排序、找不同、路线选择。
- 安全意识: 红绿灯、斑马线、施工区、乘车规则。

目标气质: commercial, cute, tactile, animated, premium toy world.

一句话定位:

> 一个会动的车车玩具世界，孩子在玩车、听车、认车、学词、闯关中完成启蒙学习。

## 2. What To Borrow From `car-word-fun`

GitHub 原型已经有几个值得保留的学习骨架:

- 选择英文: 看中文或车图，选英文。
- 选择中文: 看英文或听发音，选中文。
- 拼写单词: 根据提示补全单词。
- 限时挑战: 有计时条和分数。
- 字母启蒙: 根据首字母找单词。
- 分数、连击、历史最高分、语音朗读、结果页。

但不建议直接照搬它的呈现方式:

- 原型依赖 emoji，商业质感不足。
- 单页小测验结构过窄，缺少世界观和探索感。
- 视觉偏普通儿童网页，需要升级成“高级玩具车主题乐园”。
- 幼儿场景不应过度强调限时和连击，容易制造压力。

升级方向: 保留玩法逻辑，替换成磨砂树脂车车素材 + 场景化地图 + 明显但可控的动画反馈。

## 3. Core Creative Concept

### 车车小镇

用一个可以滚动探索的“小镇地图”承载整个网站。每个区域是一种学习游戏:

- 彩虹赛道: 路线、速度、方向、颜色。
- 字母加油站: ABC、英文发音、首字母。
- 车车收藏馆: 车型认知、分类、图鉴。
- 工程车工地: 大小、用途、配对、简单机械概念。
- 城市巴士站: 交通规则、红绿灯、公共交通。
- 修车小工坊: 车轮、车灯、车身、拼装。
- 冠军领奖台: 徽章、学习记录、亲子分享。

首页不是传统营销页，而是可玩的第一屏。用户进入后立刻看到一辆树脂小车在轨道上启动，滚动时穿过不同学习场景。

## 4. Asset Strategy

`车子素材_磨砂树脂` 里的 52 个 PNG 应作为核心商业资产使用。它们的优势是:

- 白底、干净、拟物，很适合做玩具展柜。
- 车型跨度大，适合分类学习。
- 真实车型质感强，能让页面比普通卡通更高级。

建议分类:

- 速度车: F1、跑车、AE86、GTR、R8、F40、法拉利系列。
- 城市车: 公交车、双层巴士、甲壳虫、蔚来、奥迪 A4。
- 工程车: 挖掘机、推土机、矿卡、起重机、混凝土泵车。
- 冒险车: 吉普、悍马、皮卡、越野运输车。
- 两轮/水上: 摩托、踏板车、边三轮、水上摩托。
- 特别交通: 波音 777、灰狗巴士、T 型车。

呈现规则:

- 首屏最多 3 到 5 辆车，不要一次铺满。
- 每辆车要有 hover/tap 动作: 轻微浮起、轮子转一下、阴影变化、贴纸弹出。
- 图鉴页可以用“玩具展柜”布局，每辆车在透明亚克力小格子里。
- 游戏中车图应大而清楚，避免被背景视差抢走注意力。

## 5. Game Modes

### Mode A: 车车认认看

适合年龄: 3-6 岁。

玩法: 出现一辆车，孩子从 2 到 4 个大按钮中选择名称。

学习点: 车辆名称、基础分类、图像识别。

动效:

- 正确: 车车点头、贴纸星星飞出、轨道灯亮起。
- 错误: 轻轻摇头，不扣分，不红色恐吓。
- 下一题: 车开进下一张卡片。

### Mode B: 字母加油站

适合年龄: 4-8 岁。

玩法: 看字母，选择对应英文单词或车图，例如 B -> bus。

学习点: ABC、首字母、英文发音。

动效:

- 加油枪给字母加能量。
- 字母卡片弹起，车车获得一个小闪光。
- 语音按钮做成喇叭/广播图标。

### Mode C: 车车找朋友

适合年龄: 3-7 岁。

玩法: 拖动车车到正确场景，例如挖掘机去工地、公交车去车站、赛车去赛道。

学习点: 用途分类、场景匹配。

动效:

- 拖拽时车底出现软阴影。
- 放对位置后场景局部动起来，例如工地吊臂挥一下。
- 放错位置后弹回并提示“它更适合去工地哦”。

### Mode D: 修车小工坊

适合年龄: 5-9 岁。

玩法: 给车辆装轮子、车灯、车身颜色、贴纸，完成后生成自己的车车卡。

学习点: 部件认知、顺序、颜色。

动效:

- 零件磁吸到车身。
- 安装完成时出现短暂火花/贴纸，而不是爆炸效果。
- 完成卡可进入收藏馆。

### Mode E: 交通安全小课堂

适合年龄: 4-8 岁。

玩法: 红灯停、绿灯行、斑马线过马路、施工区绕行。

学习点: 基础交通安全。

动效:

- 红绿灯是强视觉中心。
- 小车按规则停走。
- 错误反馈用暂停和解释，不做惩罚。

### Mode F: 记忆翻翻车

适合年龄: 4-10 岁。

玩法: 翻卡找相同车、找同类车、找中英配对。

学习点: 记忆、分类、词汇。

动效:

- 卡片 3D 翻转 180 度。
- 匹配成功两张卡变成同一条小轨道。
- 失败后卡片柔和翻回。

### Mode G: 小小赛道闯关

适合年龄: 6-10 岁。

玩法: 轻量路线选择，不是竞速。孩子通过回答问题、选择方向、收集字母让车到达终点。

学习点: 方向、顺序、计数、简单英文。

动效:

- 赛道作为主视差路径。
- 小车沿 SVG path 或 CSS motion path 前进。
- 每完成一关，地图上点亮一个站点。

### Mode H: 车车故事剧场

适合年龄: 3-8 岁。

玩法: 选择 3 辆车生成短故事，如“公交车带大家去公园，挖掘机修好了路”。

学习点: 语言表达、因果、角色关系。

动效:

- 纸偶剧场式横向分镜。
- 车车从左右滑入，背景层慢慢移动。
- 家长可开启朗读模式。

## 6. Parallax Presentation

### Overall Page Structure

推荐首页结构:

```text
01 Hero: 会动的车车小镇入口
02 Toy Shelf: 磨砂树脂车车收藏展柜
03 Track Journey: 横向视差学习地图
04 Game Hub: 6-8 个游戏入口
05 Learning Value: 给家长看的学习目标
06 Safety & Parent: 隐私、安全、无压力学习
07 CTA: 开始玩 / 进入收藏馆
```

### Hero Parallax

首屏做成 5 层以内:

- Layer 1: 柔和天空/浅色渐变，不抢主体。
- Layer 2: 远处圆角城市、工地、赛道牌。
- Layer 3: 中景弯曲赛道和字母路牌。
- Layer 4: 1 到 3 辆树脂车大图。
- Layer 5: 标题、主按钮、轻量进度灯。

滚动时:

- 远景移动 0.15x。
- 赛道移动 0.35x。
- 车辆移动 0.55x 并轻微上下浮动。
- 标题保持稳定，避免孩子读不清。

### Horizontal Track Journey

这是最适合商业展示的段落。用 4 到 6 个横向面板构成“车车小镇地图”:

- 第 1 站: 认车馆。
- 第 2 站: 字母加油站。
- 第 3 站: 工程车工地。
- 第 4 站: 安全斑马线。
- 第 5 站: 修车小工坊。
- 第 6 站: 徽章领奖台。

每站只讲一个核心动作，不把全部游戏说明塞满。

移动端不要强制横向滚动，改成纵向 snap 卡片。

### Detail Parallax

用于车型图鉴:

- 背景点阵慢速移动。
- 车图悬浮在前景。
- 车型标签、单词、声音按钮、收藏徽章从两侧进入。
- 同屏只动 1 到 2 个元素，避免过载。

## 7. Visual Style

方向: Resin Toy + Clay UI + Clean Commercial Landing.

不要做成廉价儿童网页。关键是“可爱但有质感”:

- 车图真实，UI 圆润。
- 背景干净，颜色明亮但不杂。
- 按钮有实体感，像软胶按钮或玩具贴纸。
- 卡片边缘圆润，但保持清爽间距。
- 动效明显，但不是所有东西都一直跳。

主色建议:

```css
:root {
  --cheche-red: #ff4b35;
  --sky-blue: #63b3ff;
  --sun-yellow: #ffd54a;
  --mint-green: #54d68a;
  --cream: #fff7e8;
  --paper: #ffffff;
  --ink: #182033;
  --soft-gray: #edf1f6;
}
```

视觉比例:

- 浅色背景 55%。
- 车图和场景色 25%。
- 红/黄/蓝功能色 15%。
- 深色文字和阴影 5%。

字体:

- 中文 UI: `Noto Sans SC`, `阿里巴巴普惠体`, `HarmonyOS Sans SC`。
- 英文/标题: `Baloo 2`, `Fredoka`, `Nunito Sans`。
- 数字和关卡: `JetBrains Mono` 少量点缀。

## 8. Commercial Quality Checklist

- 首屏必须直接可玩，不做纯品牌口号页。
- 图片是核心，所有车车都要清晰、干净、尺寸统一。
- 游戏入口要像 App Store 推荐卡一样精致，不像后台菜单。
- 每个互动都有 tap 反馈、正确反馈、完成反馈。
- 视差必须有 reduced motion 降级。
- 家长信息要安静、可信、可扫描。
- 不用 emoji 当主视觉图标，可用 SVG/Lucide 或自绘贴纸图标。
- 不做强倒计时、失败惩罚、排行榜焦虑。
- 动画优先使用 transform 和 opacity，避免布局抖动。
- 移动端优先，桌面端增强视差和展柜效果。

## 9. Recommended MVP

第一阶段不要一次做完所有玩法。建议先做 4 个高完成度模块:

1. 首页车车小镇: 视差 hero + 横向/纵向学习地图。
2. 车车认认看: 用 20 辆素材做名称选择。
3. 字母加油站: 用 15 到 20 个英文词做首字母启蒙和发音。
4. 车车收藏馆: 展示所有素材，完成游戏后点亮徽章。

第二阶段再加:

5. 交通安全小课堂。
6. 修车小工坊。
7. 记忆翻翻车。
8. 亲子学习报告。

## 10. Implementation Notes

前端动效建议:

- GSAP ScrollTrigger 或 Framer Motion: 做视差、分段 reveal、横向地图。
- CSS motion-path 或 SVG path: 做小车沿路线前进。
- Web Speech API: 英文发音，保留原型思路。
- LocalStorage: 保存徽章、历史记录、已解锁车车。
- 图片预加载: 首屏 3 到 5 张，其他素材懒加载。

数据结构建议:

```ts
type Vehicle = {
  id: string;
  zhName: string;
  enName: string;
  category: "race" | "city" | "construction" | "adventure" | "moto" | "special";
  image: string;
  ageLevel: "3-5" | "5-8" | "8+";
  facts: string[];
  soundWord?: string;
};
```

页面应该先把素材变成结构化数据，再生成游戏题目、图鉴卡、学习地图和徽章。

## 11. One Strong Direction

最推荐的方向是:

> 车车小镇 + 收藏馆 + 轻闯关学习。

不要把它做成“题库网站”。题库只是一层，外面应该包一层可探索、可收藏、可重复玩的车车世界。这样既满足幼儿学习娱乐，也能靠近成熟商业项目的完成度。
